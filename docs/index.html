SpryCSS

SpryCSS

SpryCSS aims to be a simple and customizable CSS Toolkit while being as lightweight as possible.

The goal is to remove the messy and tedious task of coming up with class names for elements that just need simple layout styles and provide basic components to help you get up and running fast, but not try to do everything which can bloat your project. If an element needs more then just the basic utility styles then that element most likely should have a custom class name and custom styles in your CSS files.

SpryCSS does not use !important. So if a SpryCSS class is not working then chances are you have already set that style in a css file and therefore you should update the style there instead of overriding it in the html.

Installation

Compatibility

SpryCss uses :has() property which is not supported by all browsers yet. Find out which browsers support it at Can I Use.
However, you can use SpryJS to add a polyfill to allow the browser to support most of the SpryCSS :has() selectors.

Using the CDN:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/ggedde/spry-css@3.0.0/dist/spry.min.css">

Using NPM and SCSS:

npm install @ggedde/spry-css

Then in your scss file import SpryCSS:

@import 'node_modules/@ggedde/spry-css/spry.scss';
Note: you may need to change the path to match your project's node_modules folder

Using Yarn and SCSS:

yarn add @ggedde/spry-css

Then in your scss file import SpryCSS:

@import 'node_modules/@ggedde/spry-css/spry.scss';
Note: you may need to change the path to match your project's node_modules folder

SpryJS Javascript Optional

SpryCSS is meant to be quick and easy to impliment without needing to add any additional dependencies. However, there are times when you need more functionality then CSS can offer. For this reason, you can include SpryJS to handle Toggles and Sliders. Toggles add additional functionality for many of its components like Accordions, Buttons, Lists, Menus, Modals, and Tabs. SpryCSS also comes with basic Sliders that have limited functionality, but you can add SpryJS for more functionality.

SpryJS

<script defer src="https://cdn.jsdelivr.net/gh/ggedde/spry-css@3.0.0/dist/spry.min.js"></script>
The SpryJS file will run the following functions on document page load.

Load Functions
Spry.loadToggles() Loads all Toggles into scope and runs all necessary features.
Spry.loadSliders() Loads all Sliders into scope and runs all necessary features.
Spry.loadLists() Loads all Lists into scope and runs all necessary features.
Spry.loadHas() Runs a polyfill to add .has-{selector} to all parent elements requiring the :has() selector.

Additional Functions
Spry.toggle(toggleObjectOrEvent, forceAction)

Toggles the class open on an element if it has been registered with data-toggle. See Toggles for more information.

toggleObjectOrEvent - Can be a valid css selector, element object, or event of element object.

forceAction - Can be close, open or toggle (default)
Spry.closeToggle(toggleObjectOrEvent)

Closes a toggle element. See Toggles for more information.

toggleObjectOrEvent - Can be a valid css selector, element object, or event of element object.
Spry.closeAllToggles()

Closes all toggle elements on the page. See Toggles for more information.

// Specify each breakpoint size
$breakpoints: (
    md: 40em,
);

// Specify each breakpoint gap size. <section> tags with get padding based on gap size.
// You must include one for each Breakpoint otherwise it will result in a Compile Error.
$gaps: (
    default: 1.25em, // required
    md: 2em,
);

// Specify each color used in color-, bg-, border-, and in accents. Each color can be bound to a theme.
$colors: (
    "grey"      : (light: #808080),                // The First Color in the array is your default color.
    "text"      : (light: #555555, dark: #eeeeee), // Variable Theme Color
    "contrast"  : (light: #333333, dark: #eeeeee), // Variable Theme Color
    "faint"     : (light: #efefef, dark: #333333), // Variable Theme Color
    "dark"      : (light: #333333),
    "light"     : (light: #efefef),
    "primary"   : (light: #1494df),

Accessing Variables in SCSS

You can access variables using map-get()

p {
    color: map-get($colors, 'primary');
    background-color: rgba(map-get($colors, 'primary'), .5);
    margin-bottom: 1rem;
    @media screen and (min-width: map-get($breakpoints, 'md')) {
        margin-bottom: .5rem;
    }
}

Breakpoints

SpryCSS is mobile first. Meaning all styles start from the smallest device then allow for changes with min-width using breakpoints. By default, SpryCSS only comes with one breakpoint md at 40em.

You can add more breakpoints in the SCSS Variables, but keep in mind that the more breakpoints you add the larger the compiled file size will be.

SCSS Variable 
$breakpoints: (
    md: 40em,
)

Used In
Borders Borders allow breakpoints. ex <div class="b-0 md-b-2">No Border on Mobile</div>
Gaps Gaps uses breakpoints to define initial margin in <hr> and initial padding in <section>
Grid Grid uses gaps in CSS Grid and CSS Flexbox grids
Sizing Spacing allows breakpoints. ex <div class="sm md-lg">Small on Mobile, Large on Medium</div>
Spacing Spacing allows breakpoints. ex <div class="p-0 md-p-2">No Padding on Mobile</div>
Layout Utility Classes Layout Utility Classes allow breakpoints. ex <div class="none md-block">Hide on Mobile</div>

Gaps

Default gaps (aka gutters) are automatically adjusted based on your breakpoints. If you add more breakpoints in your SCSS Variables you will also need to add the corrisponding Gap.

SCSS Variable 
$gaps: (
    default: 1.25em, // required
    md     : 2em,
)
* You need to make sure the gaps match the breakpoints otherwise you will receive a SCSS compile error.

Activator Attributes
data-toggle="{selector}" This creates the activator, assigning it to the selector element.
{selector} can be:
  • Any valid css selector
  • next which assigns it to the next sibling.
  • hover which assigns it to the next sibling, but only on hover. Requires class hoverable.
  • empty which assigns it to itself.
data-toggle-action="{action}" Assigns the toggle action.
{action} can be:
  • open
  • close
  • toggle (default) Toggles between open and close.
data-toggle-close="{selector}" Closes Toggle(s) by a valid css selector.
{selector} can be:
  • Any valid css selector
  • empty which closes the closest .open ancestor.
{selector} can start with {*} to be used with "closest".
<button data-toggle-close="{.tabs} .open">
This will close all .open elements within the closest .tabs ancestor from the activator.
data-toggle-timeout="{n}" Automatically closes the toggle after the timeout in milliseconds.
data-toggle-escapable Allows the toggle to close when clicking outside the toggle element or by pressing the ESC key.
data-toggle-dismissible Allows the toggle to close when clicking inside the toggle element on an actionable child element.

    These are basic lightweight sliders. If you are looking for something more robust, you might want to check out Swiper or Glide

    You can add loading="lazy" to the images, but it may be a little choppy when scrolling on some browsers. The Slider will detect the lazy loading images and pre load them on the next and previous slides when using SprySliderJS. If you want a smooth transition it is best to load all the images initially.

<div class="slider" data-snap>

        <img id="no-js-slide2" src="/placeholder-2.jpg" loading="lazy">
        <img id="no-js-slide3" src="/placeholder-3.jpg" loading="lazy">
    </div>
</div>

    data-snap Forces stopping on each slide even when using grid layout. This is not forced with momentum scrolling.

    • action - Pause autoplay while hovering over a link, button or making a text selection within slider.
