SpryCSS

SpryCSS

SpryCSS aims to be a simple and customizable CSS Toolkit while being as lightweight as possible (Only 10kb gzipped).

The goal is to remove the messy and tedious task of coming up with class names for elements that just need simple layout styles and provide basic components to help you get up and running fast, but not try to do everything which can bloat your project. If an element needs more then just the basic utility styles then that element most likely should have a custom class name and custom styles in your CSS files.

SpryCSS does not use !important. So if a SpryCSS class is not working then chances are you have already set that style in a css file and therefore you should update the style there instead of overriding it in the html.

No Javascript Required

SpryCSS is meant to be quick and easy to impliment without needing to add any additional dependencies or any javascript. SpryCSS comes with Toggles which uses hidden checkboxes and radio buttons to impliment javascript like functionality for many of its components like Accordions, Modals, and more. SpryCSS also comes with basic Sliders that don't need any javascript.

However, the functionality is limited and may not be suitable for all projects especially with strict accessibility requirements. So SpryCSS does have an optional small javascript library available to extend the functionality for these basic components if needed.

Getting Started and Installation

Compatibility

SpryCss uses :has() property which is not supported by all browsers yet. Find out which browsers support it here

Using the CDN:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/ggedde/spry-css@3.0.0/dist/spry.min.css">

Using NPM and SCSS:

npm install @ggedde/spry-css

Then in your scss file import SpryCSS:

@import 'node_modules/@ggedde/spry-css/spry.scss';
Note: you may need to change the path to match your project's node_modules folder

Using Yarn and SCSS:

yarn add @ggedde/spry-css

Then in your scss file import SpryCSS:

@import 'node_modules/@ggedde/spry-css/spry.scss';
Note: you may need to change the path to match your project's node_modules folder

Overriding SCSS Variables:

To override the Spry Variables just declare the variables before you import the file. 

@import 'your_variables.scss';
@import 'node_modules/@ggedde/spry-css/spry.scss';

Custom Build:

Cherry pick the components you need and leave out the ones you don't need for a smaller build.

SCSS Variables 

// Specify each breakpoint size
$breakpoints: (
    md: 40em,
);

// Specify each breakpoint gap size. <section> tags with get padding based on gap size.
// You must include one for each Breakpoint otherwise it will result in a Compile Error.
$gaps: (
    default: 1.25em, // required
    md     : 2em,
);

// Specify each color
$colors: (
    "default"  : #888888,
    "light"    : #efefef,
    "dark"     : #333333,
    "primary"  : #1494df,
    "secondary": #df8a22,
);

// Specify each type tag and class like <h3> and .h3 etc.
$types: (
    body: (font-family: Helvetica, font-size: clamp(.9em, 3vw, 1.0em), font-weight: 300, color: var(--contrast), line-height: 1.4),
    h1  : (font-size: 3.2em, font-weight: 400, line-height: 1.20),
    h2  : (font-size: 2.3em, font-weight: 300, line-height: 1.25),
    h3  : (font-size: 2.0em, font-weight: 300, line-height: 1.30),
    h4  : (font-size: 1.4em, font-weight: 300, line-height: 1.35),
    h5  : (font-size: 1.1em, font-weight: 300, line-height: 1.5),
    h6  : (font-size: 1.0em, font-weight: 300, line-height: 1.4),
    a   : (color: var(--primary), text-decoration: none),
);

// Specify each size that applies font-size and all elements using "em".
$spacing: (
    0: 0em,

Accessing Variables in SCSS

You can access variables using map-get()

p {
    color: map-get($colors, 'primary');
    background-color: rgba(map-get($colors, 'primary'), .5);
    margin-bottom: 1rem;
    @media screen and (min-width: map-get($breakpoints, 'md')) {
        margin-bottom: .5rem;
    }
}

Icons

SpryCSS comes with a small set of custom icons. You can either use <i> or class .icon

Need More Icons?

You can use whatever ican package or icon webfont you want, but icon webfonts can greatly increase the size of your projects. A recommended alternative is to use direct svg code within your project. This produces the smallest file size while producing high resolution icons and allowing you the most flexibility to style the icons within your css.

A great option for this is Material Design Icons. You can search for icons and view the SVG code by clicking on the Code dropdown and click on 'View SVG'. Then copy and paste the code in your html.

If your using Visual Studio Code you can install the extension vscode-materialdesignicons-svg that provides code snippets to quickly add the SVG code.

Breakpoints

SpryCSS is mobile first. Meaning all styles start from the smallest device then allow for changes with min-width using breakpoints. By default, SpryCSS only comes with one breakpoint md at 40em.

You can add more breakpoints in the SCSS Variables, but keep in mind that the more breakpoints you add the larger the compiled file size will be.

SCSS Variable
$breakpoints: (
    md: 40em,
)

Used In
Gaps Gaps uses breakpoints to define initial margin in <hr> and initial padding in <section>
Grid Grid uses gaps in CSS Grid and CSS Flexbox grids
Spacing Spacing allows breakpoints. ex <div class="p-0 md-p-2">No Padding on Mobile</div>
Layout Utility Classes Layout Utility Classes allow breakpoints. ex <div class="none md-block">Hide on Mobile</div>

Gaps

Default gaps (aka gutters) are automatically adjusted based on your breakpoints. If you add more breakpoints in your SCSS Variables you will also need to add the corrisponding Gap.

SCSS Variable
$gaps: (
    default: 1.25em, // required
    md     : 2em,
)
* You need to make sure the gaps match the breakpoints otherwise you will receive a SCSS compile error.

Grid

Container Classes
grid CSS Grid of columns auto width. Uses variable Gaps for spacing
grid-{n}00 CSS Grid of columns using minmax() for column widths in px's. {n} can be 1-12
row CSS Flexbox Row of columns auto width. Uses variable Gaps for spacing
{breakpoint}-columns-{n} {n} can be 1-12, auto, fixed

* fixed and auto only apply to row
{breakpoint}-g-{n} Gap for columns within grid or row

See Spacing for available {n} values.

Column Classes
{breakpoint}-col-{n} {n} can be 1-12, auto, fixed

* Custom gaps will not always work correctly with row when setting specific column sizes. It works best if you specify all breakpoint column sizes or use it with grid instead

Toggles

Toggles are used to open and close content without needing any Javascript.
Can be used for Dropdowns, Modals, Menus, Accordions, etc.
<!-- Hidden Toggle -->
<input type="checkbox" class="toggle" />
<p>Toggle Content</p>
<!-- Button with Icon Toggle -->
<input type="checkbox" class="toggle button icon down" />
<p>Toggle Content</p>
<!-- Text Toggle -->
<label class="toggle">Toggle Handle
    <input type="checkbox" />
</label>
<p>Toggle Content</p>

Tags
input The trigger for the toggle. Can be of type checkbox or radio
label (Optional) wrapper for a toggle to allow more content.
.toggle + * Any element immediately following a toggle is hidden.

Toggle Classes
toggle Activates a Toggle and hides the immediate following sibling
hoverable Allows toggle to activate on hover
active Forces Toggle Active
* All toggles that are :checked or has :checked are considered active.

Sibling Classes
{breakpoint}-open Forces sibling open even when toggle is not active.
Can be used with or without breakpoints
collapse Positions all child elements absolute.
